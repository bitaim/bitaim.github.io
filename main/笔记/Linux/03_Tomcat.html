# Tomcat

  Tomcat是SpringBoot内嵌的Servlet容器，也是最常用的Web容器。Tomcat8开始采用NIO的网络模型，之前采用BIO的网络模型。

# 参数

server:
  port: 5085
  tomcat:
    # 连接超时时间，默认值为60000ms
    # 客户端与服务端建立连接后，服务端在超时时间内等待客户端的消息，如果客户端一直不发消息，则服务端会主动关闭该连接
    connection-timeout: 700ms
    # Tomcat与客户端可以建立连接的最大数量，默认值为10000
    max-connections: 2000
    accept-count: 100
    # Tomcat工作线程的最大数量，默认值为200
    threads:
      max: 200

1、acceptCount参数

  当客户端与服务端建立连接之后，就会进入到操作系统级别的队列中，队列长度由acceptCount决定。

  该参数受到Linux系统参数/proc/sys/net/core/somaxconn的影响，操作系统中该参数默认值为128。如果acceptCount参数大于操作系统的操作，则以操作系统为主。

  Tomcat会监听操作系统的一些事件，来处理这些建立好的连接。max-connections参数是Tomcat所能建立连接的最大数量。当Tomcat建立的连接数一满，并且操作系统级别的队列一满，在Windows系统下，就会拒绝新的连接。而在Linux系统中，还有一层三次握手的队列，再然后是操作系统中完成握手的队列。简单的认为，tomcat能够建立连接的最大数量为max-connections参数与acceptCount参数的和即可。

# 开发

1、定制化SpringBoot内置Tomcat

  通过对Tomcat的长连接进行配置，提升请求性能。

@Configuration
public class TomcatConfig implements WebServerFactoryCustomizer<ConfigurableServletWebServerFactory> {
    @Override
    public void customize(ConfigurableServletWebServerFactory factory) {
        ((TomcatServletWebServerFactory) factory).addConnectorCustomizers(connector -> {
            Http11NioProtocol http11NioProtocol = (Http11NioProtocol) connector.getProtocolHandler();
            // 一个长连接支持的最大请求数量，默认值为100
            // 当参数为 -1 时，表示不限制请求数量
            // 当参数为 1  时，表示禁用长连接
            http11NioProtocol.setMaxKeepAliveRequests(5);
            // 建立长连接后，无请求时的长连接保持时长
            // 默认值等于connectiontimeout的时长，也就是60000毫秒
            http11NioProtocol.setKeepAliveTimeout(60000);
        });
    }
}

2、Tomcat的长连接

  （1）Http1.1默认支持长连接，当客户端与服务端建立连接后，后续的请求可以复用该连接，避免了频繁建立连接的损耗。

  （2）客户端发送Http请求时，请求头携带Connection: keep-alive表示希望建立一个长连接。

  （3）如果服务端同样支持长连接，服务端在响应数据后不会立马断开连接，尝试复用连接。

  （4）当达到长连接最大请求数量时，服务端会响应头中携带Connection: close，表示要客户端关闭长连接。

  （5）实验如下。

# 启动Tomcat进程
[root@iz2ze4oq2onbrfblgura6tz jar]# netstat -anp | grep 8080
tcp        0      0 0.0.0.0:8080            0.0.0.0:*               LISTEN      7351/java

# 客户端与服务端建立连接后，显示 ESTABLISHED状态（如果该连接是长连接，则保持KeepAliveTimeout的时间）
# 当有后续请求时，重置持续时间
[root@iz2ze4oq2onbrfblgura6tz jar]# netstat -anp | grep 8080
tcp        0      0 0.0.0.0:8080            0.0.0.0:*               LISTEN      7351/java           
tcp        0      0 172.17.115.0:8080       120.11.173.121:18935    ESTABLISHED 7351/java

  （6）阿里巴巴开发手册中提到，在Linux服务器上通过变更/etc/sysctl.conf文件去修改缺省值net.ipv4.tcp_fin_timeout = 30，该参数在默认情况下是60秒。

  高并发服务器下建议调小TCP协议的time_wait超时时间，在高并发访问下，服务器端会因为处理time_wait的连接数太多，可能无法建立新的连接，所以需要调小此等待值。

3、Time_Wait连接

  当Tomcat不支持长连接时，客户端请求服务端，每一个请求结束后，客户端都会主动断开连接，主动断开连接的一方进入TIME_WAIT状态。

  当Tomcat支持长连接时。服务端与客户端建立了一个持久连接，持续时间为30秒钟，客户端在期间也没有新的请求进来，那么服务端会在30秒后主动断开连接。正常情况下，期待的是客户端请求结束之后主动断开连接，但不能保证客户端不主动关闭的情况，或者非法连接的情况。

# 运维

1、如何安装Tomcat

https://tomcat.apache.org/download-80.cgi (opens new window)

  在官网中下载对应的软件包，Windows选择64-bit Windows zip (pgp, sha512)，Linux选择tar.gz (pgp, sha512)。解压后即安装成功。

2、常用命令

  在解压后的Tomcat/bin目录下，是常用的命令脚本。

  启动Tomcat，Windows下是startup.bat，Linux下是startup.sh。

  停止Tomcat，Windows下是shutdown.bat，Linux下是shutdown.sh。

3、访问Tomcat时，如何直接访问到某个项目

  （1）修改tomcat/webapps/ROOT目录下的index.jsp文件，配置如下。当访问ip:port时，Tomcat会自动重定向到ip:port/项目名。

<%
java.text.SimpleDateFormat sdf = new java.text.SimpleDateFormat("yyyy");
request.setAttribute("year", sdf.format(new java.util.Date()));
request.setAttribute("tomcatUrl", "https://tomcat.apache.org/");
request.setAttribute("tomcatDocUrl", "/docs/");
request.setAttribute("tomcatExamplesUrl", "/examples/");
# 新增如下一行
response.sendRedirect("/web");
%>

  （2）修改tomcat/conf目录下的server.xml文件，配置如下。当访问ip:port时，可以直接访问到项目名下，在地址栏上忽略了项目名。

<Host name="localhost"  appBase="webapps"
            unpackWARs="true" autoDeploy="true">
    <!-- 新增如下一行 --> 
    <Context path="/" docBase="web" />
</Host>
