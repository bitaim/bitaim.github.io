# DSL 语法

  Elasticsearch7.X的不同版本下，请求和响应会有些许差异，但整体是不会变的。

# 字段类型

1、字符串类型

  text类型会对内容进行分词。

  keyword类型不会对内容进行分词，会当作一个整体。

2、数值类型

  long、integer、short、byte、double、float、half_float、scaled_float

3、布尔类型

  boolean

4、二进制类型

  binary，开发中用的很少。

5、范围类型

  integer_range、float_range、date_range、long_range、double_float

  比如我们有一个job_index，其中有一个字段为职业生涯周期，我们将其定义为integer_range类型。比如java开发的职业生涯周期为{"gte":20,"lte":35}，大于等于20小于等于35。

  我们用查询21岁可以做的工作，就可以通过"term":{"age":21}搜索到java开发这条数据。

6、日期类型

  由于json中没有date类型，所以es通过识别字符串是否符合format定义的格式来判断是否为date类型。

  下面的格式就满足es的格式。

2022-01-01
2022/01/01 12:10:30
从1970年1月1日0点开始的秒数（毫秒数）

# 复杂数据类型

数组类型Array，ES中没有专门的数组类型，直接使用[]定义即可，数组中所有的值必须是同一种数据类型，不支持混合数据类型的数组。 字符串数据，数字数组，Object对象数组

对象类型，对象类型内部可能还有对象。

{ "query": { "term": { // 索引方式通过 . 的方式 "address.location.province": "广州" } } }

# 文档

  ES7之后，索引的type类型被固定为_doc。

  当新增文档时，如果发现Elastcisearch中没有该索引，在默认情况下会自动创建索引。 在真实情况下，自动创建的索引的mapping往往与业务情况不符合，还是手动创建的好。

PUT http://127.0.0.1:9200/_cluster/settings

{
    "persistent": {
        // 关闭自动创建索引
        "action.auto_create_index": false
    }
}

请求行：http://127.0.0.1:9200/employee_index/_doc，Elasticsearch自动生成文档的主键。

# term查询

  term（词条查询是单词级别查询，通常用于结构化的数据，比如number、data、keyword等，而不是text。Elasticsearch官方警告，避免使用term对text类型的字段查询。

  单词级别的查询，不会对输入的内容进行分词，直接在相应字段的倒排索引上精确查找。

# 1、term

请求方式：GET

请求行：http://127.0.0.1:9200/employee_index/_search

类比：select * from employee_index where idNumber = '130922199909093611' limit 0,10

请求体：

{
    "query": {
        "term": {
            "idNumber": "130922199909093611"
        }
    },
    "form": 0,
    "size": 10
}

{
    "query": {
        "term": {
            "idNumber": {
                "value": "130922199909093611"
            }
        }
    },
    "form": 0,
    "size": 10
}

{
    "query": {
        // 不关心相关度算分
        "constant_score": {
            "filter": {
                "term": {
                    "idNumber": "130922199909093611"
                }
            }
        }
    },
    "form": 0,
    "size": 10
}

# 2、terms

请求方式：GET

请求行：http://127.0.0.1:9200/employee_index/_search

请求体：

{
    "query": {
        "terms": {
            "idNumber": [
                "130922199909093611",
                "130922199909093612"
            ]
        }
    }
}

# match查询

  match查询的字段的类型为text类型，根据其中设置的search_analyzer搜索分词器对客户端的输入进行分词，再去字段的倒排索引中检索数据。

# 1、match

  在单个字段内进行全文检索。

请求方式：GET

请求行：http://127.0.0.1:9200/employee_index/_search

请求体：

{
    "query": {
        "match": {
            "name": "张三"
        }
    }
}


{
  "query": {
    "match": {
      "name": {
        "query": "云",
        // 当我们输入多个关键词时，可能返回的结果许多都是只满足一个关键词的
        // 可以通过该参数使返回的结果更加精准，避免返回相差甚远的结果
        "minimum_should_match": 2
      }
    }
  }
}

# 2、multi_match

  在多个字段内进行全文检索。

请求方式：GET

请求行：http://127.0.0.1:9200/employee_index/_search

请求体：

{
    "query": {
        "multi_match": {
           "query": "张三",
           "type":"best_fields",
           // name的权重更高
           // "fields": ["name^2", "areaNo"]
           "fields": ["name", "areaNo"],
           // 到时候百度查一下吧
           "tie_breaker": 0.3,
           "minimum_should_match": 2
        }
    }
}

# 3、match_phrase

  短语（词组）匹配。返回的结果中必须包含整个短语，并不能被分割。可以理解为 name like %张三%。

请求体：

{
    "query": {
        "match_phrase": {
           "name": "张三"
        }
    }
}

{
    "query": {
        "match_phrase": {
           "name": {
               "query": "张三",
               // 允许移动一次，匹配到结果
               // 张某三这样着也可以被查询出来了
               "slot": 1
           }
        }
    }
}

# 4、match_phrase_prefix

  前缀匹配，理解为 name like 张%

请求体：

{
    "query": {
        "match_phrase_prefix": {
           "name": "张"
        }
    }
}

# 3、查询全部数据

{
    "query": {
        "match_all": {

        }
    }
}

# 提升全文检索结果的精确度

  当我们搜索数据时，可能会同时输入多个关键词A、B，这样在返回结果中就会有三种类型的数据，一种是包含A关键词，一种是包含B关键词，另一种是包含A、B两个关键词。

  而我们这里说的，提升精确度，比如只让结果返回包含A、B两个关键词的结果。

GET employee_index/_search

// 包含 “云” 或 “聪” 的数据都会被检索出来
{
  "query": {
    "match": {
      "name" :"云聪"
    }
  }
}

// 包含 “云” 和 “聪” 的数据才会被检索出来
{
  "query": {
    "match": {
      "name" :{
        "query": "云聪",
        "operator": "and"
      }
    }
  }
}

// 数据中满足75%的内容才会被匹配出来
{
  "query": {
    "match": {
      "name" :{
        "query": "云聪大小",
        "minimum_should_match": "75%"
        // "minimum_should_match": 1（支持满足个数）
      }
    }
  }
}

# boost权重控制

  搜索名字中包含李的员工，要是名字中又包含宁要优先搜索出来。

{
    "query": {
        "bool": {
            "must": {
                "match": {
                    "name" :"李"
                }
            },
            "should": [
                {
                    "match": {
                        "name": {
                          "query": "宁",
                          // 重设权重值
                          "boost": 1
                        }
                    }
                }
            ]
        }
    }
}

# range范围查询

  查找指定字段在指定范围内的日期、数字、字符串的文档。

# 1、多范围条件查询

  查询年纪大于18岁，身高大于等于170，30天内加入的李姓员工。

  注意此处使用的是must字句，会计算文档的相关度分值，如果不需要相关度分值，请使用filter字句。

GET employee_index/_search

GET employee_index/_search
{
    "query":{
        "bool":{
            "must":[
                {
                    "match": {
                      "name": "李"
                    }
                },
                {
                    "range":{
                        "age":{
                            "gt":18
                        }
                    }
                },
                {
                    "range":{
                        "height":{
                            "gte":170
                        }
                    }
                },
                {
                    "range": {
                        "joinDate": {
                            //"gt": "2021-07-24||-82d"（别出现空格）
                            //"gt": "2021-07-24||-2y"（别出现空格）
                            //"gte": "now-30d"  当前时间减少30天
                            //"gte": "now-30y"  当前时间减少30年
                            // 不能按月减少，不管事
                            "gt": "2021-07-25||-30d"
                        }
                    }
                }
            ]
        }
    }
}

# 1、查找1999年到2022年之间的人

{
    "query": {
        "range": {
            "birthday": {
                "gte": "01/01/1999",
                "lte": "2022",
                "format": "dd/MM/yyyy||yyyy"
            }
        }
    }
}

{
    "query":{
        "constant_score": {
            "filter": {
                "range": {
                    "age": {
                        "gte": 18,
                        "lt": 35
                    }
                }
            }
        }
    }  
}

# sort排序

  匹配名字中包含张三，然后按照年龄倒序，身高正序的方式排列。

{
    "query": {
        "match": {
            "name": "张三"
        }
    },
    "sort": [
        {
            "age": {
                "order": "desc"
            }
        },
        {
            "height": {
                "order": "asc"
            }
        }
    ]
}

# 布尔查询

{
    "query": {
        "constant_score": {
            "filter": {
                "bool": {
                    // job为Java或者IOS
                    "should": [
                        {
                            "term": {
                                "job": "Java"
                            }
                        },
                        {
                            "term": {
                                "job": "IOS"
                            }
                        }
                    ],
                    // position必须为
                    "must": [
                        {
                            "term": {
                                "position": "河北省"
                            }
                        },
                        {
                            "term": {
                                "gender": "男"
                            }
                        }
                    ],
                    "must_not": {
                        "term": {
                            "position": "河北省"
                        }
                    }
                }
            }
        }
    }
}
