# 介绍

  官方地址，https://www.elastic.co/cn/elasticsearch/。

索引（index），一个索引可以理解成一个数据库（表），在ES中是唯一的。

类型（type），在ES7.X版本之后，移除type的概念，并默认每个index的type为**_doc**。

映射（mapping），定义每个字段的类型，相当于定义表结构。

文档（document），索引中的一条数据。

# refresh

  当编辑一条数据时，我们希望立刻就能搜索到，但是真实情况不是这样的。

  其实当编辑一条文档时，文档被放入到一个buffer缓冲区，此时是搜索不到的。默认是每隔一秒钟会将缓冲区的数据刷新到索引中，才可以被检索到。

# 别名

  对于ES来说，为了适应新的业务需求，可能就需要对原有的索引做一些修改。比如对某个字段做调整，甚至是重建索引，而做这些操作时，可能会对业务造成影响，甚至是停机调整等问题。由此，es提供了索引别名来解决这些问题。

  比如有一个旧的索引为 employee_index，我们需要改变其中一个字段的类型，那么就需要重建索引。

  （1）首先为employee_index创建一个别名，别名对外使用。

  （2）创建一个新索引，employee_index_now。

  （3）重建索引。将数据复制到新索引。

POST /_reindex

{
    "source": {
        "index": "employee_index"
    },
    "dist": {
        "index": "employee_index_now"
    }
}

  （4）在新索引中，创建同样的别名。在删除旧索引的别名。最后删除旧索引即可。

1、什么是全文搜索引擎

  搜索引擎是指通过某些关键字来获取我们想要的信息。

2、使用场景

  比如常用搜索网站，百度、谷歌。比如常用的购物网站，淘宝、京东。都离不开搜索引擎的使用。

百度搜索引擎、淘宝京东搜索等。适用于大数据量的全文搜索。

  Elasticsearch以文档的方式存储数据，基于这种文档数据结构，可以提供复杂的索引，全文检索，分析聚合等功能。文档（Document）用JSON数据格式来表达。

  无论存储的数据结构嵌套多少层的对象，只需要一个文档就可以记录。不需要像关系型数据库，需要拆成多个表，在关联起来查询存储。

在ES5.2版本之后，如果将字段type设置为text类型，默认会创建两个Field。一个是field本身（分词），另一个是field.keyword（不分词）。

# constant_score查询

这种查询忽略分数的计算，性能高，还能使用缓存

{
    "query":{
        "constant_score":{
            "filter":{
                "term":{
                    "value":"Book"
                }
            }
        }
    }
}

英文分词一般会先转成小写，存储到倒排索引。因为term不会分词，所以通过Book去查询可能获取不到结果，book就可以。





# 模糊查询

对输入内容最多做一次调整，再去查询。
比如数据中title是 neverending,
neverendiog 做一次调整 neverending，就可以查看该数据了

{
    "query":{
        "fuzzy":{
            "title":{
                "value":"neverendiog",
                "fuzziness":1
            }
        }
    }
}

must是所有的条件必须满足，对相关度进行算分。 must_not是所有的条件都不能满足，并且不对相关度算分。 filter是所有的条件必须满足，对不会对相关度算分。 should是满足其中一个条件，并且算分。
