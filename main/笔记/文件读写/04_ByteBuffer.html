# ByteBuffer

  字节缓冲区ByteBuffer底层维护一个字节数组，提供了丰富的API进行读写。其中还有另外三个属性很重要，position、limit、capacity，是实现读写的关键指针。

  字节缓冲区ByteBuffer又分为HeapByteBuffer堆内内存和DirectByteBuffer堆外内存两种。

  字节缓冲区ByteBuffer，通过缓冲区进行读写，减少了系统调用的次数，是读写高性能的主要原因。

# 堆内内存

// 分配4byte的堆内内存
ByteBuffer byteBuffer = ByteBuffer.allocate(4);

  该对象被分配在堆空间中。

1、堆内内存读写时的隐藏关卡

  当使用堆内内存进行网络IO或者磁盘IO进行读写操作时，都会临时创建一块堆外内存。作为堆内内存的数据的中转站，与操作系统进行读写操作。交互就像这样HeapByteBuffer <-> DirectByteBuffer <-> os。

  在使用ByteBuffer进行读写时，是直接基于内存地址进行读写的。是为了躲避在读写过程中，可能产生GC而导致的读写错误问题。

  因为在发生GC时，会进行内存整理，势必会导致对象的地址发生变化。如果在基于旧的地址进行读写时，就会发生错误。

  因此需要临时创建一块堆外内存来做中转。因为JVM是无法管理堆外内存的，因此读写数据是安全的。

# 堆外内存

// 分配4byte的堆外内存
ByteBuffer byteBuffer = ByteBuffer.allocateDirect(4);

  其中的对象是分配在堆空间中的，但其中引用的一大块内存是在堆外内存中的。而如果该对象实例进入了老年代，可能很长时间内都不会发生老年代的GC，导致堆外内存不能被释放。在分配新的堆外内存空间中，可能出现堆外内存溢出的问题。

堆外内存的回收

  当创建DirectByteBuffer实例时，会生成一个Cleaner实例。如下图所示。

// DirectByteBuffer中的一个成员变量
private final Cleaner cleaner;

DirectByteBuffer(int cap) {                   // package-private

    super(-1, 0, cap, cap);
    boolean pa = VM.isDirectMemoryPageAligned();
    int ps = Bits.pageSize();
    long size = Math.max(1L, (long)cap + (pa ? ps : 0));
    Bits.reserveMemory(size, cap);

    long base = 0;
    try {
        base = unsafe.allocateMemory(size);
    } catch (OutOfMemoryError x) {
        Bits.unreserveMemory(size, cap);
        throw x;
    }
    unsafe.setMemory(base, size, (byte) 0);
    if (pa && (base % ps != 0)) {
        // Round up to page boundary
        address = base + ps - (base & (ps - 1));
    } else {
        address = base;
    }
    // // 创建DirectByteBuffer时，同时创建一个Cleaner实例
    cleaner = Cleaner.create(this, new Deallocator(base, size, cap));
    att = null;
}

Deallocator负责回收堆外内存。如下图所示。

private static class Deallocator
        implements Runnable
{

    private static Unsafe unsafe = Unsafe.getUnsafe();

    private long address;
    private long size;
    private int capacity;

    private Deallocator(long address, long size, int capacity) {
        assert (address != 0);
        this.address = address;
        this.size = size;
        this.capacity = capacity;
    }

    public void run() {
        if (address == 0) {
            // Paranoia
            return;
        }
        unsafe.freeMemory(address);
        address = 0;
        Bits.unreserveMemory(size, capacity);
    }

}

Reference类中有一个后台线程ReferenceHandler，当发生GC时，就会回收这些堆外内存。

static boolean tryHandlePending(boolean waitForNotify) {
    Reference<Object> r;
    Cleaner c;
    try {
        synchronized (lock) {
            // pending 是一个链表，当发生GC时，JVM的垃圾回收线程就会给链表赋值
            if (pending != null) {
                // 下面几行代码 是将链表第一个元素出队
                r = pending;
                // 'instanceof' might throw OutOfMemoryError sometimes
                // so do this before un-linking 'r' from the 'pending' chain...
                c = r instanceof Cleaner ? (Cleaner) r : null;
                // unlink 'r' from 'pending' chain
                pending = r.discovered;
                r.discovered = null;
            } else {
                // The waiting on the lock may cause an OutOfMemoryError
                // because it may try to allocate exception objects.
                if (waitForNotify) {
                    lock.wait();
                }
                // retry if waited
                return waitForNotify;
            }
        }
    } catch (OutOfMemoryError x) {
        // Give other threads CPU time so they hopefully drop some live references
        // and GC reclaims some space.
        // Also prevent CPU intensive spinning in case 'r instanceof Cleaner' above
        // persistently throws OOME for some time...
        Thread.yield();
        // retry
        return true;
    } catch (InterruptedException x) {
        // retry
        return true;
    }

    // Fast path for cleaners
    if (c != null) {
        // 调用Cleaner的clean方法，clean方法就会同步调用Deallocator的run方法
        // 完成堆外内存回收的工作
        c.clean();
        return true;
    }

    ReferenceQueue<? super Object> q = r.queue;
    if (q != ReferenceQueue.NULL) q.enqueue(r);
    return true;
}

# 常用API

public class ByteBufferTest {
    public static void main(String[] args) {
        // 申请4字节大小的堆内内存
        // 默认是写模式
        ByteBuffer byteBuffer = ByteBuffer.allocate(4);
        // [pos=0 lim=4 cap=4]
        // postion 代表要写入的位置，postion < limit
        // limit 代表不可写的位置
        // capacity 代表容量
        System.out.println(byteBuffer);
        
        // 写入三字节数据
        byteBuffer.put((byte)1);
        byteBuffer.put((byte)2);
        byteBuffer.put((byte)3);
        // [pos=3 lim=4 cap=4]
        System.out.println(byteBuffer);
        
        // 切换到读模式
        byteBuffer.flip();
        // position 要读取的位置，position < limit 
        // limit 不可读的位置
        // capacity 容量
        // [pos=0 lim=3 cap=4]
        System.out.println(byteBuffer);
        // 读取一个字节的数据
        byte a = byteBuffer.get();
        // [pos=1 lim=3 cap=4]
        System.out.println(byteBuffer);
    }
}
