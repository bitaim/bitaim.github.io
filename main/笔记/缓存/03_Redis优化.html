# Redis优化

# 注意事项

  1. 避免使用大value，会导致网络相关的故障问题。
  2. 对热key做处理，避免集群被打爆，可能会导致连锁反应。最好加一个熔断保护机制。

# 批处理

# 1、mget

  使用Redis自带的mget批量查询命令来优化循环get单条查询命令。

redisTemplate.opsForValue().multiGet(keys);

# 2、hmset

  将多次hset插入合并成hmset一次插入。

# 3、pipeline

  使用pipeline管道来做批处理，可以大大提升性能。

redisTemplate.executePipelined(new RedisCallback<String>(){
    @Override
    public String doInRedis(RedisConnection connection) throws DataAccessException {
        for (int i = 0; i < 10000; i++) {
            connection.lPush("queue".getBytes(), String.valueOf(i).getBytes());
        }
        // StringRedisConnection src = (StringRedisConnection)connection;
        return null;
    }
});

  当Redis采用集群部署时，pipleline管道中的所有命令涉及的key必须应用在同一个slot上，否则就会报错。

  在Redis集群中，是通过CRC16(key)%16384来计算slot的位置的。Redis提供了hashtag特性，可以根据用户指定key中的部分字段计算slot位置。将key中的部分字段加上花括号，user:{1229098765}。

# API

# SCAN

  当使用keys <keyword>命令时，会阻塞命令执行线程，导致服务出现卡顿，生产环境禁止使用keys命令。

  此时应当使用scan <keyword>命令，可以做到无阻塞的取出指定模式的key列表，但有一定的重复概率，需要做一次去重，另外耗时相比keys会长。

# LIST

  一般使用list结构作为队列，rpush生产消息，lpop消费消息。当lpop没有消息的时候，

  当时在项目中，需要调用淘宝的改价接口，该接口对于店铺的请求进行了限流。一分钟最多请求120次，每秒最多2次。我们创建了线程来延时消费redis中的异步队列，当没有数据时进行阻塞（bloop命令）。

# ZSET

  使用sortedset做延时队列，用时间戳作为score，通过zrangebyscore指令获取N秒之前的数据进行处理。

  当时在项目中，有一个间隔1分钟的爬虫任务，首先去从Redis获取N秒内的数据，根据这些数据爬取相关的信息。同时计算下一次数据的爬取时间（热点产品5分钟后再次爬取，否则20分钟后再次爬取），再次存入Redis。

# GEO


public class GeoService {
    
    // 上传位置
    public void add(Point point, String userId) {
        redisTemplate.opsForGeo().add("user_geo", new RedisGeoCommands.GeoLocation<>(userId, point));
    }

    // 附近的人
    public GeoResults<RedisGeoCommands.GeoLocation> near(Point point) {
        Distance distance = new Distance(100, RedisGeoCommands.DistanceUnit.METERS);
        Circle circle = new Circle(point, distance);
        RedisGeoCommands.GeoRadiusCommandArgs geoRadiusCommandArgs = RedisGeoCommands.GeoRadiusCommandArgs.newGeoRadiusArgs().includeDistance().limit(5);
        GeoResults<RedisGeoCommands.GeoLocation> userGeo = redisTemplate.opsForGeo().radius("user_geo", circle, geoRadiusCommandArgs);
        return userGeo;

        // userGeo.getContent().getName(); 用户ID
        // userGeo.getContent().getValue(); 距离
    }
}
