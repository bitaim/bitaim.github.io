# ThreadLocal

  当数据的生命周期较短时，比如在一个线程的级别下。并且线程之间的数据互相独立，互不影响。就可以采用ThreadLocal，来做到线程封闭。

  比如在ReentrantReadWriteLock中，使用ThreadLocal来记录每个线程加共享锁的次数。

  比如在前置拦截器中，解析JWT然后将用户信息存入ThreadLocal。可以在请求的生命周期内任意位置获取到用户的信息。关键在后置拦截器移除用户信息，防止内存泄漏。

public class Thread implements Runnable {

    // Thread类 与 ThreadLocal类的关联
    ThreadLocal.ThreadLocalMap threadLocals = null;

}

# ThreadLocal设置

public void set(T value) {
    // 获取 执行set(T)方法的 当前线程
    Thread t = Thread.currentThread();

    // 获取当前线程的 ThreadLocalMap 
    ThreadLocalMap map = getMap(t);
    if (map != null) {
        // 直接赋值
        map.set(this, value);
    } else {
        // 初始化 map，并赋值
        createMap(t, value);
    }    
}


ThreadLocalMap getMap(Thread t) {
    // ThreadLocalMap类 存储在 Thread类中
    return t.threadLocals;
}

void createMap(Thread t, T firstValue) {
    t.threadLocals = new ThreadLocalMap(this, firstValue);
}

# ThreadLocal获取

public T get() {
    Thread t = Thread.currentThread();
    ThreadLocalMap map = getMap(t);
    if (map != null) {
        ThreadLocalMap.Entry e = map.getEntry(this);
        if (e != null) {
            @SuppressWarnings("unchecked")
            T result = (T)e.value;
            return result;
        }
    }
    // 初始化ThreadLocalMap，与set方法相同，只不过初始值为null
    return setInitialValue();
}

# ThreadLocal移除

public void remove() {
    ThreadLocalMap m = getMap(Thread.currentThread());
    if (m != null)
        m.remove(this);
}

  当使用完线程后，务必移除。

# ThreadLocalMap

static class ThreadLocalMap {

    // Entry 是 弱引用的子类
    static class Entry extends WeakReference<ThreadLocal<?>> {
        
        Object value;

        Entry(ThreadLocal<?> k, Object v) {
            // Entry 中的key以弱引用的方式指向 ThreadLocal类的实例
            super(k);
            value = v;
        }

    }

    ThreadLocalMap(ThreadLocal<?> firstKey, Object firstValue) {
        table = new Entry[INITIAL_CAPACITY];
        int i = firstKey.threadLocalHashCode & (INITIAL_CAPACITY - 1);
        // ThreadLocalMap 通过 Entry 存储数据
        table[i] = new Entry(firstKey, firstValue);
        size = 1;
        setThreshold(INITIAL_CAPACITY);
    }

    // 底层是一个Enrey数组
    private Entry[] table;
    // 初始化容量为16
    private static final int INITIAL_CAPACITY = 16;

}

1、为什么使用弱应用的方式

  当把ThreadLocal类的实例置为null后，由于没有任何强引用指向该实例，所以ThreadLocal类的实例能被GC顺利回收。

  如果以强引用的方式指向ThreadLocal类的实例，那么就会与Entry存在强引用而无法被回收，造成内存泄漏。除非所有存在强引用的线程结束被回收，那么map也会被回收。

2、内存泄漏问题

  通过弱引用的方式解决了ThreadLocal类的实例的回收问题。虽然Entry类的key为null，但是value还是被强引用着（Thread->map->Entry->value），但是value却不会被访问到了，因此存在内存泄漏问题。除非线程结束了，map才可以被回收。

# InheritableThreadLocal

  把线程本地变量传递给子线程。

public class ThreadLocalTest {

    private static ThreadLocal<String> threadLocal = new ThreadLocal<>();
    private static ThreadLocal<String> inheritableThreadLocal = new InheritableThreadLocal<>();

    public static void main(String[] args) {
        threadLocal.set("主线程");
        inheritableThreadLocal.set("子线程");

        new Thread(() -> {
            System.out.println("sub thread local value: " + threadLocal.get());     // null
            System.out.println("sub inheritable thread value: " + inheritableThreadLocal.get());    // 子线程
        }, "subThread").start();

        System.out.println("main thread local value: " + threadLocal.get());    // 主线程
        System.out.println("main inheritable thread value: " + inheritableThreadLocal.get());   // 子线程
    }

}
