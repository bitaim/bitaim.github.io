# 异步

  通常会通过线程池的方式，来异步处理某些非核心逻辑。如果要获取线程池的结果，要怎么做呢？

public interface ExecutorService {
    // execute 是没有返回结果的
    void execute(Runnable command);
    // 下面是三种带返回结果的方法
    <T> Future<T> submit(Callable<T> task);
    <T> Future<T> submit(Runnable task, T result);
    Future<?> submit(Runnable task);
}

  对于这种没有返回值的，使用execute(Runnable task)即可，无需构造一个返回对象。

  这种submit提交任务的方式，其实会将异常封进Future。如果不主动调用future.get()方法，那么异常会被吞掉。然后当我们调用该方法时，又会阻塞当前线程。Future的实现不是一种好的方式，对于异步编程甚是难受。真正的异步编程应该是自动回调，而不用阻塞主线程。

  在JDK8迎来了异步编程的春天，出现了CompleteFuture。

# CompletableFuture

# 1、ExecutorCompletionService

public class FutureTest {

    private static ExecutorService executorService = Executors.newFixedThreadPool(3);

    public static void main(String[] args) throws InterruptedException, ExecutionException {

        ExecutorCompletionService<String> completionService =
                new ExecutorCompletionService<>(executorService);

        completionService.submit(() -> {
            TimeUnit.SECONDS.sleep(15);
            return "15秒的任务";
        });

        completionService.submit(() -> {
            TimeUnit.SECONDS.sleep(5);
            return "5秒的任务";
        });

        completionService.submit(() -> {
            TimeUnit.SECONDS.sleep(10);
            return "10秒的任务";
        });

        for (int i = 0; i < 3; i++) {
            // 可以获取到最快执行完的任务
            String result = completionService.take().get();
            System.out.println(result);
        }

    }

}

# 2、CompletableFuture

public class CompleteFutureTest {

    private static ExecutorService executorService = Executors.newFixedThreadPool(10);

    public static void main(String[] args) {

        // （1） runAsync 不支持返回值
        CompletableFuture<Void> noResponseFuture = CompletableFuture.runAsync(() -> {
            System.out.println("执行无返回值业务");
        }, executorService);

        // (2) supplyAsync 支持返回值
        CompletableFuture<String> hasResponseFuture = CompletableFuture.supplyAsync(() -> {
            System.out.println("第一轮处理");
            return "success";
        }, executorService);

        // （3）异步任务完成后的回调方法
        // （3.1） 仅接收上一阶段异步处理的结果，没有新的返回值
        hasResponseFuture.thenAccept(response -> {
            System.out.println("第二轮处理");
        });
        // （3.2） 接收上一阶段异步处理的结果，并返回新的返回值
        hasResponseFuture.thenApply(response -> {
            System.out.println("第二轮处理");
            return response + "_success";
        });

        // （4）对异常处理

    }

}
