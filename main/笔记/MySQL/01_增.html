    # 主键

    # 1、常用主键

    1. 自增主键。
    2. UUID。
    3. 分布式主键。比如雪花算法、开源框架Leaf（美团）。

    # 2、为什么UUID不适合做主键

      （1）在性能方面考虑。因为UUID不是单调递增的，当插入数据时很大概率都是向不同的数据页插入，写入一个数据页最终就需要刷到磁盘上，无疑增加写IO的损耗。还有概率引发页分裂，导致移动其他的数据行，使空间利用率下降。相邻的两个利用率很低的数据页，还可能引发页合并，又会影响性能。而自增的主键，每次插入都是追加操作，性能很高。

      （2）在存储方面考虑。由于非主键索引的叶子节点存储的是主键ID的值。UUID作主键占用32字节，长整型long作主键占用8字节。对于相同量的数据，UUID作为主键的方式会占用更大的磁盘空间。

    # 3、自增主键不一定连续

      在MySQL5.7及之前版本，InnoDB存储引擎的自增值（AUTO_INCREMENT）保存在内存中，并没有持久化。在MySQL8之后才提供自增值持久化的能力。

      MySQL5.7版本。在MySQL实例重启后，第一次打开表的时候，会去寻找记录中主键的最大值,然后将 最大值 + 1 作为这个表的自增值（下一次插入数据时使用的主键ID）。

      举例。首先插入一条主键为10的数据，此时自增值为11。然后删除主键为10的数据。最后重启MySQL实例，此时恢复出的该表的自增值为10。

      MySQL8.0版本。会将自增值的变更记录在redo log中，重启时依赖redo log恢复自增值。

      主键不连续的情况，主要由以下几种原因。

      （1）插入数据时，如果自增列的值为null、0、未指定时，那么会获取一个自增值并更新（+1）。如果由于冲突插入失败，那么就出现了主键不连续的情况。

      （2）插入数据时，如果指定了主键具体值并且大于等于自增值，那么会更新自增值为具体值+1。如果具体值小于自增值，那么不会更新自增值。由于指定主键值，也会造成主键不连续的情况。

      （3）事务回滚，也会造成主键不连续的情况。

      （4）对于普通的 insert语句，自增锁在申请之后就会马上释放。对于insert ... select这样的批量插入语句，自增锁要在事务结束之后才会释放，如果该事务执行过慢，将会严重影响其他事务的插入操作。MySQL中申请自增ID的策略。语句在执行过程中，第一次申请会分配一个，第二次申请会分配2个，第三次申请会分配4个。申请的自增ID可能会出现浪费的情况，也会造成主键不连续的情况。

    package com.bitaim.study.project.redis.mapper;

import com.bitaim.study.project.redis.entity.Property;
import org.apache.ibatis.annotations.Param;
import java.util.List;

public interface PropertyMapper {

    /**
     * 插入
     * @param property
     */
    void insert(Property property);

    /**
     * 批量插入
     * @param propertyList
     */
    void insertBatch(@Param(value = "propertyList") List<Property> propertyList);

}


    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.bitaim.study.project.redis.mapper.PropertyMapper">

    <!-- 单条添加 并回显主键 -->
    <insert id="insert" keyProperty="id" useGeneratedKeys="true" parameterType="com.bitaim.study.project.redis.entity.Property">
        INSERT INTO property(name, value)
        VALUES (#{name}, #{value})
    </insert>

    <!-- 批量添加 -->
    <insert id="insertBatch" keyProperty="id" useGeneratedKeys="true" parameterType="com.bitaim.study.project.redis.entity.Property">
        INSERT INTO property(name, value)
        VALUES
        <foreach collection="propertyList" item="property" separator=",">
            (#{property.name}, #{property.value})
        </foreach>
    </insert>

</mapper>
