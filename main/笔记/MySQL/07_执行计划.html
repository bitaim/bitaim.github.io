    # 执行计划

      查看执行计划是SQL调优的重要手段。通常会观察以下几个指标。

    # type

      （1）const：表示使用唯一性索引进行查询，最多返回一行数据。举例：

    -- 主键索引查询
select * from user where id = 41;
-- 手机号唯一索引查询
select * from user where phone = '15230795085';

      （2）eq_ref：表示使用唯一性索引进行关联查询，最多返回一行数据。举例：

      （3）ref：表示使用非唯一索引进行关联查询，可以匹配到多个符合条件的行。举例：

    select * from address where user_id = 41 
select * from user left join address on user.id = address.user_id where user.id = 41

      （4）range：表示使用索引进行了范围查询，避免了全表扫描。举例：

    -- 使用的操作符有：!=、=、>、>=、between、in等。
select * from user where id >= 41;
-- != 索引失效，对主键索引不管事
select * from user where id != 41; -- 还是range级别
-- != 索引失效，对于唯一索引、普通索引是失效的

      （5）index：表示全索引扫描，比ALL的效率要高一些。举例：

    -- 查询时使用了覆盖索引
select id from user;
select phone from user;
-- 使用了索引进行排序
select * from user order by id;

      （6）ALL：表示需要进行全表扫描，是性能最差的类型。举例：

    select * from user

    # extra

    （1）Using filesort。说明无法利用索引进行排序，需要利用排序算法进行排序，性能差。

（2）Using temporary。说明使用临时表来保存中间结果，查询之后会删除临时表。

（3）Using index。说明使用了覆盖索引，性能好。

（4）Using where。说明使用了where进行条件过滤。

（5）Using join buffer。在联表查询时，表的连接条件没有用到索引，需要有一个连接缓冲区来存储中间结果。

    # Using filesort 案例

      当Extra字段出现Using filesort字段时，表示需要排序。MySQL会给每个线程分配一块内存用于排序，称为sort_buffer。

    1、语句执行流程

    explain select name, age, province from user where province = '河北省' order by name limit 1000

      （1）首先初始化一块sort_buffer内存。

      （2）在 province索引 中寻找第一个满足条件的主键ID。

      （3）再到主键索引中寻找完整的行数据，取三个字段的值放入sort_buffer中。

      （4）重复（2）、（3）步骤，直到不满足条件为止。

      （5）对sort_buffer中的数据按照name字段进行排序。

      （6）将排序后的前1000行数据返回给客户端。

    2、临时文件辅助排序

      如果整个排序的过程，可以在内存中完成，那很好。如果完成不了，则需要使用到外部排序。

      取决于sort_buffer_size的参数大小，该参数决定每个线程分得排序缓冲区的内存大小。如果要排序的数据量小于该块内存，则可以在内存中完成。如果大于这块内存，就需要利用磁盘临时文件辅助排序（归并排序算法）。

    3、优化

      参数max_length_for_sort_data是MySQL中专门控制用于排序的行数据长度的参数。如果单行的长度超过这个值，MySQL就会选择另外一个算法。

      首先把max_length_for_sort_data参数设置为16，新的算法放入sort_buffer的字段只有name和id两个字段。这样sort_buffer中能放的数据量就会增多，但由于缺少了两个字段，排序后的结果不能直接返回给客户端了。

      （1）首先初始化一块sort_buffer内存。

      （2）在province索引中找到第一个满足province='河北省'条件的主键ID。

      （3）再到主键索引中寻找完整的行数据，取name、id的值，放入sort_buffer中。

      （4）重复（2）、（3）步骤，直到不满足条件为止。

      （5）对sort_buffer中的数据按照字段name进行排序。

      （6）取排序后的前1000行，并按照id的值到原表中取出province、name、age三个字段返回给客户端。

      这种算法，使得sort_buffer中存储的行数变多了，因此使用的临时文件也变少了。但多了回表查询的过程。

      其实，最好的办法还是在排序字段上增加索引，就可以避免使用排序。

    # Using temporay

      表示需要创建临时表。而临时表并不全是内存表，是由temp_table_size参数决定的，参数默认值是16MB。如果临时表超过了该值，内存表就会转变为临时表。磁盘临时表使用的引擎默认是InnoDB，是由参数internal_tmp_disk_storage_engine控制的。
