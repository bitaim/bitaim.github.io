# 其他问题

# 存储引擎

1、Memory

  如果表使用的是Memory存储引擎，那么这是一张内存表，当MySQL重启的时候数据都会丢失。优势是数据保存在内存中，读写速度快。内存表不支持行锁，只支持表锁。

  建议所有表使用InnoDB引擎，唯一一个例外的场景是使用Memory存储引擎做临时表，性能最好。

2、InnoDB

  InnoDB是MySQL5.5.5版本之后默认的存储引擎，是目前最常用的存储引擎。InnoDB存储引擎支持行锁，能够支撑高并发，支持事务。数据存储于聚簇索引，有成熟的数据库结构，分库分表，读写分离，主备切换，都是基于InnoDB存储引擎而来的。

3、MyISAM

  MyISAM不支持事务，表级锁。但索引文件和数据文件分开，因为内存里可以缓存更多的索引，对查询的性能更好，适用于少量插入，大量查询的需求。适合用于报表系统。

# 临时表

  临时表也是有存储引擎的。如果使用InnoDB存储引擎或者MyISAM存储引擎，那么数据是写到磁盘上的，称为磁盘临时表。如果使用Memory存储引擎，那么数据是存储在内存中的，称为内存临时表。临时表最常用的还是内存临时表，因为性能更高。

1、如何创建临时表

create temporary table temp_user(id int primary key, name varchar(32), age int, index (name))engine=memory;
insert into temp_user select * from tb_user where b>=1 and b<=2000;
select * from tb_group join temp_user on (tb_group.b = temp_user.b);

2、临时表的特性

  1、临时表只能被创建它的事务访问，对其他事务不可见。在事务结束时，会自动删除临时表。

  2、临时表可以与普通表重名。事务内同时存在同名的临时表和普通表，增删改查操作是应用在临时表上的。

3、临时表的用途

  临时表经常会被用于复杂查询的优化过程中，分库分表的跨库查询就是一个典型的应用场景。

4、MySQL在哪些地方使用了临时表

  当使用union语句时，取两个子查询结果的交集，交集放在临时表中去重。而使用union all时，则不需要使用临时表。

# 编码格式

  MySQL的utf-8编码格式只支持3字节的数据，当数据库需要存储emoji表情时，就可能出现java.sql.SQLException: Incorrect string value: ‘\xF0\x9F\x92\x94’ for column ‘name’ at row 1错误。

  utf8mb4编码是utf8编码的超集，兼容utf8，并且能存储4字节的表情字符。 采用utf8mb4编码的好处是：存储与获取数据的时候，不用再考虑表情字符的编码与解码问题。修改数据库的编码为utf8mb4，修改my.cnf（windows为my.ini）配置文件，需要添加以下配置。然后重启mysql即可。

[client] 
default-character-set = utf8mb4 
[mysql] 
default-character-set = utf8mb4 
[mysqld] 
character-set-client-handshake = FALSE 
character-set-server = utf8mb4 
collation-server = utf8mb4_unicode_ci 
init_connect='SET NAMES utf8mb4'

# MySQL结构

  大体来说，M有SQL可以分为Server和存储引擎两部分。结构如下图所示。

  连接器：负责与客户端建立连接、获取权限、维护和管理连接，权限验证。建立好的连接，可以通过show processlist指令查看，Command列为Sleep，表示是一个空闲连接，空闲连接长时间没有操作，就会被自动断开，由参数wait_timeout控制，默认是8小时。

  分析器：对SQL语句进行解析，语法分析和验证语句，比如表是否存在，列是否存在等。

  优化器：执行计划生成，索引的选择。可以通过explain指令来获取执行计划，针对性进行调优。

  执行器：在执行之前，还要判断一下是否有对表的操作权限，然后调用存储引擎接口，将符合条件的数据放入结果集，最终返回结果集给执行器。

  存储引擎：存储数据，提供读写接口。

# 常用命令

1、修改字段部分内容

update 表名 SET 列名 = REPLACE( 新的列名, '被替换内容', '替换内容' );

2、修改字段

alter table 表名 MODIFY column 列名 decimal(10,3);

3、添加字段

alter TABLE 表名 add 列名 VARCHAR(20) NOT NULL comment '注释';

alter TABLE 表名 add 列名 VARCHAR(20) NOT NULL comment '注释' AFTER 列名;在某个字段后面追加字段。

# 数据库挂了

  连接MySQL，执行SELECT 1;如果正常返回，证明数据库进程还在，但不能证明数据库是没问题的。

# 隐式类型转换

  当主键为varchar类型，而传入数值型主键时，会发生强制类型转换。会将索引树的主键转换为数值型与传入参数比较。

# 隐式字符编码转换

  当两个表进行关联查询时，如果驱动表的字段为utf8mb4类型，而被驱动表的字段为utf8类型。为了避免数据在转换过程中由于截断导致数据错误，都是按数据长度增加的方向进行转换。也就是说被驱动表的字段需要进行转换，由utf8->utf8mb4。而导致全表扫描。
