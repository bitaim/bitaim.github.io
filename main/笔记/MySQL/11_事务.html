# 读未提交原理

  在读未提交隔离级别下，存在行锁。但是读取数据时，每次读取的都是存储引擎中的最新数据。也就是出现了脏读的问题。

# 读已提交原理

  为了解决脏读的问题，引入了MVCC（Multiversion Concurrency Control）多版本并发控制。 MVCC只在读已提交和可重复读两个隔离级别下工作。MVCC依赖于隐藏字段、Read View、Undo log实现。

1、一致性视图 ReadView

  一致性视图也被称为快照。ReadView是一种数据结构，主要包含以下几部分。

  （1）trx_ids。负责存储未提交的事务id。

  （2）min_trx_id。指 trx_ids 中最小的事务id。

  （3）max_trx_id。系统中生成过的事务 ID最大值 + 1。

  （4）creator_trx_id。存储当前事务的id。

  在RC隔离级别下，事务开启之后，每次执行select语句，都会重新创建一个快照。

  在RR隔离级别下，事务开启之后，第一次执行select语句，才会创建一个快照。整个事务均使用该快照。

2、可见性判断

  如何判断记录行的版本是否可见。取要读取的记录行的版本记为trx_id。

  （1）如果 trx_id < min_trx_id，代表该记录行在当前事务开启前就已经提交了，数据可见。

  （2）如果 trx_id >= max_trx_id，代表该记录是由未来开启的事务修改的。不可见，跳到步骤4。

  （3）如果 min_trx_id =< trx_id < max_trx_id。判断是否是当前事务自己修改的，如果是那可见。如果不是判断是否在活跃的事务列表内，如果不在，那么可见。如果在，不可见，跳到步骤4。

  （4）通过该记录行的回滚指针，找到历史版本的数据，将事务id赋给 trx_id，回到步骤1重新开始判断。

  当查询数据时，首先要判断记录行的版本是否在 trx_ids 中，如果在那就看不到。并且只能看到小于等于当前事务的数据版本。每次查询时，刷新快照，即可以看到已提交的数据。

# 可重复读原理

  在开启事务后，通过第一次查询生成一致性视图，整个事务基于该视图来保证可重复读。

  快照读，通常使用的select语句，就是基于MVCC的快照读。

  当前读，能够读到所有提交的事务的最新值。比如update、delete语句都是当前读，特殊的有select ... for update也是当前读，这三种方式加的都是排它锁，会对扫描到的行加上行锁。保证同一时刻只有一个事务可以操作该数据，保证数据的一致性。

  幻读是在当前读下才会出现，专指新插入的行。为了解决幻读问题，可重复读隔离级别下引入了间隙锁。间隙锁主要解决了两次当前读的可重复读问题。

  行锁只能锁住扫描到的行，却无法锁住插入操作。因此使用间隙锁锁住了扫描到的行之间间隙，来阻止插入操作。

现有主键 1、7、15对应的三条数据

（1）主键索引存在。

select * from user where user_id = 7 for update; // 对记录7加行锁。

select * from user where user_id > 7 for update; // 添加间隙锁(7,15)，(15,+)，对记录15加行锁。

select * from user where user_id >= 7 for update; // 添加间隙锁(7,15)，(15,+)，对记录7，记录15加行锁。

（2）主键索引不存在。

select * from user where user_id = 5 for update; // 添加间隙锁(1,7)

select * from user where user_id > 5 for update; // 添加间隙锁(1,7)，(7,15)，(15,+)，对记录7，记录15加行锁。

select * from user where user_id >= 5 for update; // 添加间隙锁(1,7)，(7,15)，(15,+)，对记录7，记录15加行锁。

（3）唯一索引存在。

select * from user where id_card = 'aaa' for update; // id_card唯一索引将 aaa 加锁，此时另一条aaa的数据时，会被阻塞，而不是报重复。

（5）普通索引存在。

有三个班级，一班，七班，十五班。

select * from user where class_id = 7 for update; // 首先要锁住7班的所有行（主键索引），然后对班级索引加间隙锁。添加间隙锁[1，15)。

select * from user where class_id > 7 for update; // 首先要锁住15班的所有行（主键索引），然后对班级索引加间隙锁。添加间隙锁[7,15)，[15,++)

select * from user where class_id >= 7 for update; // 首先要锁住7班，15班的所有行（主键索引），然后对班级索引加间隙锁。添加间隙锁[1,7)，[7,15)，[15,++)
