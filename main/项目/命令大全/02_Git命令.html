# Git命令

# 创建本地仓库

  进入要创建本地仓库的目录，执行git init命令即可。

# 完整的提交流程

  通过git status查看本地仓库文件情况。

  通过git add .将当前目录下的文件添加到暂存区。

  通过git commit -m <提交内容>将暂存区的文件提交到本地仓库。

  通过git pull命令拉取远端仓库代码。如果存在冲突，冲突解决后通过git add .和git commit -m <提交内容>命令提交到本地仓库。

  通过git push命令将本地仓库推送到远端仓库。

# 工作区

  通过git checkout -- <file>即可恢复工作区的文件到最近的一次Add动作或者最近的一次Commit动作。

# 暂存区

  通过git add .指定当前目录的文件添加到暂存区。

  通过git add [dir]指定目录下文件添加到暂存区。

  通过git add [file1] [file2]指定多个文件文件添加到暂存区。

  通过git rm --cached [file1]移除暂存区的文件。

# get reset

  这个是用来做代码回退的命令。代码回退分三个模式，其中mixed是默认模式，下面分别介绍。

  通过git log命令查看Commit记录，找到要回退的版本号。

  （1）通过git reset --soft <版本号>，撤销Commit动作。

  （2）通过git reset --mixed <版本号>，撤销Commit动作和Add动作。

  （3）通过git reset --hard <版本号>，撤销Commit动作和Add动作，将工作目录的改动也完全恢复。

  如果回退的版本仅仅发生在本地仓库的话，是很简单的。如果回退的某个版本已经提交到远程仓库，那么就有些特殊了。因为回退到历史某个版本后，本地仓库的版本号小于远程仓库的版本号。

  处理本地代码后，需要强制Push本地仓库到远程仓库。

# 分支

  通过git checkout -b test01 master，基于 master本地分支 创建 test01本地分支。

  通过git push origin test01，推送 本地分支test01 到 远程仓库。如果远程仓库没有test01分支，那么会自动创建该分支。

  通过git branch --set-upstream-to=origin/test01 test01，第一次推送本地分支到远程仓库后，需要将本地test01分支与远程test01分支进行关联，方便后续代码的拉取git pull和推送git push。

  通过git branch -d test01，即可删除本地分支。
