# Git安装

1、官方地址下载：Git下载地址

2、设置环境变量

  1. 右键电脑空白处，点击 Git Bash Here
  2. 设置全局变量用户名称 git config --global user.name "yuncong"
  3. 设置全局变量用户邮箱 git config --global user.email "yuncong@163.com"

3、SSH

  为了更加方便快捷的上传拉取代码，通常都会使用SSH的方式。

  1. ssh-keygen -t rsa -C "yuncong@163.com"
  2. 进入C:\Users\firstlaptop.ssh，打开id_rsa.pub，复制公钥
  3. 打开GitLab或者其他代码仓库，选settings，左侧SSH Keys，粘贴进入，title自动填充你的邮箱，然后点击Add Key

4、提交一次代码

  1. git status 检查本地代码的状态
  2. git add . 将本地代码提交到暂存区
  3. git status 再次检查本地代码的状态
  4. git commit -m "备注" 将暂存区的代码提交到本地仓库
  5. git pull 在远程仓库拉取代码，拉取代码之前要先commit一遍
  6. git push origin master 解决冲突，然后提交到本地仓库，最终推送代码到远程仓库

5、回退一次版本

  1. git log 查看所有提交记录，↑ ↓ 滚动，按 q 退出。git log --pretty=oneline，优雅的方式查看历史版本
  2. git reset --hard <commit id> 根据版本号回退
  3. git reset --hard head^ 回退上一次版本，git reset --hard head^^ 回退上上次版本，git reset --hard head~5 回退5次版本
  4. git reflog查看更多记录，包含提交、回退记录

6、分支命令

  1. git branch 查看所有分支
  2. git branch demo-2 创建分支
  3. git branch -d demo-2 删除本地分支
  4. git checkout demo-2 切换分支

7、标签命令

  1. git tag --list 查看所有标签
  2. git tag t1.0.0 当前分支创建标签
  3. git tag -d t1.0.0 删除本地分支
  4. git push origin t1.0.0 推送标签
  5. git push origin :t1.0.0 删除远程仓库标签

8、合并命令

  1. git merge demo-1 将demo-1的代码合并到当前分支

9、撤销命令

  1. 修改了UserController，发现改错了，需要撤销对其的修改 git checkout UserController.java

10、其他指令

  1. git init 初始化本地仓库
  2. git remote add origin git@github.com:bitaim/photo.git 关联远程仓库

# MySQL安装

1、官方地址下载ZIP Archive版本并解压：MySQL下载地址

2、设置环境变量

在Path中设置：解压后的 绝对路径\bin

3、创建一个my.ini的文件（mysql的配置文件）

# mysql5.7以后不再提供配置的模板文件,需要新建一个。
# 新建my.ini配置文件，并简单修改几项内容：
[mysql]
# 设置mysql客户端默认字符集
default-character-set=utf8
[mysqld]
#设置3306端口
port = 3306
# 开启binlog
log-bin=mysql-bin
binlog-format=Row
server_id=100
# 设置mysql的安装目录
basedir=D:\DevelopSoftware\mysql-5.7.28-winx64
# 设置mysql数据库的数据的存放目录
datadir=D:\DevelopSoftware\mysql-5.7.28-winx64\data
# 允许最大连接数
max_connections=200
# 服务端使用的字符集默认为8比特编码的latin1字符集
character-set-server=utf8
# 创建新表时将使用的默认存储引擎
default-storage-engine=INNODBs

4、以管理员身份运行CMD命令窗口,并执行相应操作

  Ctrl + R 搜索 cmd 右键管理员运行

y: 就可以切换到y盘
cd Y:\mysql5.7.28\bin
mysqld --defaults-file=D:\DevelopSoftware\mysql-5.7.28-winx64\my.ini --initialize --user=mysql --console

  注：此处若遇到遇到 MSVCR120.dll文件丢失错误 ，解决方法如下： 下载 VC redist packages for x64，下载完成，点击运行即可。

5、安装mysql服务

mysqld --install MySQL --defaults-file=D:\DevelopSoftware\mysql-5.7.28-winx64\my.ini

6、启动mysql服务

net start mysql

7、首次连接需要修改root密码

mysql -uroot -p

mysql> flush privileges;

# Tomcat参数配置

  ConnectionTimeout默认20秒。处理请求的超时机制，是tomcat的自我保护机制。如果处理请求长时间没有结束，那么会认为是请求超时，建议降低。

  maxThreads默认200，tomcat处理的最大线程数（处理能力），也就是吞吐量。建议增加，但不是越大越好。理想的线程数量=（1+代码阻塞时间/代码执行时间）*CPU数量。实际情况是跑起代码，压测环境进行调试。不断调整线程数，将CPU打到80~90%的利用率。

  maxConnections默认nio1万，进入tomcat的请求量是不能大于这个数的，是tomcat的连接数限制，与线程池的队列类似。最好比预期的最高并发数量大20%，反正是堆积到Tomcat的work处理线程池中，占些内存而已。

  acceptCount默认100，超过maxConnections的数量之后，就会进入操作系统的一个等待区域，是一个socket参数，在操作系统底层的一个队列中维护请求，默认是100，建议增加。Tomcat能承受的最大连接数 = acceptCount + maxConnections。
