# Linux常用软件

# JDK8安装

  1、下载JDK安装包并上传到服务器，指令tar -zxvf jdk-8u251-linux-x64.tar.gz解压缩，将解压后的目录移动到/usr/local目录下。

  2、指令vi /etc/profile，修改环境变量，在末尾添加即可，配置如下。

export JAVA_HOME=/usr/local/jdk1.8.0_251
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export PATH=$PATH:$JAVA_HOME/bin

  3、指令source /etc/profile，刷新配置文件使其立即生效。

  4、确认Java环境是否安装成功，指令java -version，显示版本号即可。

# Redis安装

  1、下载Redis安装包并上传到服务器，指令tar -zxvf redis-5.0.8.tar.gz解压缩。

  2、Redis编译需要安装gcc环境，指令yum install -y gcc。

  3、指令cd /usr/local/redis-5.0.8进入解压后目录，指令make进行编译。如果抛出zmalloc.h:50:31: 致命错误：jemalloc/jemalloc.h：没有那个文件或目录错误，执行指令make MALLOC=libc，再次执行指令make进行编译。

  4、指令cd src切换目录，执行指令make install PREFIX=/usr/local/redis安装命令即可。

1、常用配置

  复制解压后的目录中的redis.conf到/usr/local/redis目录下，然后修改redis.conf配置文件。

  1、后台运行模式，daemonize yes。

  2、允许远程访问。关闭保护模式protected-mode no，将bind 127.0.0.1注释。

  3、添加密码，requirepass <密码>。

2、常用命令

  1、启动Redis，指令cd /usr/local/redis切换目录，指令bin/redis-server redis.conf即可启动成功。

  2、关闭Redis，指令cd /usr/local/redis切换目录，指令bin/redis-cli shutdown即可关闭Redis。

# Nginx安装

  1、下载Nginx安装包，地址为http://nginx.org/en/download.html，并上传到服务器，指令tar -zxvf nginx-1.18.0.tar.gz解压缩。

  2、安装依赖环境，yum -y install gcc pcre-devel zlib-devel openssl openssl-devel。

  3、切换目录cd nginx-1.18.0，编译./configure --prefix=/usr/local/nginx，安装make && make install。

1、常用命令

  1、启动Nginx，指令cd /usr/local/nginx切换目录，指令sbin/nginx即可启动成功。

  2、重启Nginx，指令cd /usr/local/nginx切换目录，指令sbin/nginx -s reload即可重启成功。

  3、快速停止Nginx，指令cd /usr/local/nginx切换目录，指令sbin/nginx -s stop即可停止成功。

  4、优雅停止Nginx，指令cd /usr/local/nginx切换目录，指令sbin/nginx -s quit即可停止成功。

  5、重建日志文件，指令cd /usr/local/nginx切换目录，指令sbin/nginx -s reopen即可停止成功。

2、负载均衡

3、openresty

  使用Openresty代替Nginx，Openresty中整合了相关lua脚本，有利于nginx开发工作者开发。

  • 从 http://openresty.org/en/ 左侧的download中下载相关tar.gz的linux安装包
  • 上传到linux服务器后，解压 tar -zxvf openresty-1.13.6.2.tar.gz
  • cd openresty-1.13.6.2，进入目录后，安装相关依赖 yum -y install pcre-devel openssl-devel gcc curl
  • ./configure 编译，然后 make&&make install 安装
  • 进入目录 cd /usr/local/openresty/nginx
  • 启动：sbin/nginx -c conf/nginx.conf
  • 无缝重启：修改配置后直接 sbin/nginx -s reload

# MySQL安装

  1、下载MySQL安装包并上传到服务器，指令tar -zxvf mysql-5.7.30-linux-glibc2.12-x86_64.tar.gz解压缩。

  2、修改解压后的文件名mv mysql-5.7.30-linux-glibc2.12-x86_64 mysql5.7.30。

  3、修改配置vi /etc/my.cnf，配置如下。

[client]
port=3306
socket=/tmp/mysql5.7.30/mysql.sock
[mysqld]
port=3306
user=mysql
character_set_server=utf8mb4
collation-server=utf8mb4_general_ci
socket=/tmp/mysql5.7.30/mysql.sock
basedir=/usr/local/mysql5.7.30
datadir=/usr/local/mysql5.7.30/data
symbolic-links=0

skip-grant-tables
[mysqld_safe]
log-error=/var/log/mysql5.7.30/mysql.log
pid-file=/var/run/mysql5.7.30/mysql.pid

!includedir /etc/my.cnf.d

  4、创建用户useradd -m mysql，创建组groupadd mysql。

  5、将新建的与mysql相关的文件夹的权限全部修改，chown -R mysql:mysql mysql5.7.30。

  6、初始化数据库/bin/mysqld --initialize --user=mysql --basedir=/usr/local/mysql5.7.30 --datadir=/usr/local/mysql5.7.30/data，将临时密码复制到某处保存。

  7、第一次启动mysql时需要初始化配置，需要切换用户su mysql，指令/usr/local/mysql5.7.30/support-files/mysql.server start即可启动mysql，mysql.server stop即可停止mysql。切换回root用户也可以正常启动关闭mysql。

  8、修改环境变量。指令vi /etc/profile，在末尾加入export PATH=$PATH:/usr/local/mysql5.7.30/bin:/usr/local/mysql/lib，指令source /etc/profile使配置生效。

  9、修改密码。指令vi /etc/my.cnf修改配置，将skip-grant-tables注释。root用户重新启动mysql。指令mysql -uroot -p登录mysql，输入临时密码，登录成功后。指令set password='123456';即可修改密码。

1、开放远程访问

  数据库安装成功后，通过navicat并不能直接远程连接，会抛出1130错误，is not allowed to connect to this server。

  1、首先在安装mysql的服务器执行mysql -uroot -p登录，选择数据库use mysql;，执行select host from user where user = 'root';，可以看到host的值为localhost，也就是只允许本机访问。

  2、指令update user set host = '%' where user ='root';，Host设置了“%”后便可以允许远程访问。

  3、flush privileges;，刷新权限使配置生效即可连接。

2、MySQL环境变量

  配置好环境就可以在任意目录，使用mysql -uroot -p登录命令了。

MYSQL_HOME=/usr/local/mysql5.7.30
PATH=$PATH:$MYSQL_HOME/bin
export MYSQL_HOME PATH

3、开启binlog

server_id=100
log-bin=/usr/local/mysql5.7.30/binlog/mysql-bin
binlog-format=Row

4、常见错误

  有一次为了开启binlog，再登录时出现这个错误，MySQL... ERROR! The server quit without updating PID file。百度许久，未找到解决方案。最关键的还是要去看日志，mysqld: File '/usr/local/mysql5.7.30/binlog/mysql-bin.index' not found (Errcode: 13 - Permission denied)结果是目录没有权限。这个目录是为了存储binlog而新创建的，突然想到了mysql的文件权限都应该是mysql，所以调整目录权限chown mysql:mysql binlog。

# Docker安装

  0、Docker CE支持64位Centos7系统，并且内核版本不低于3.10。可通过uname -a查看内核版本。

  1、创建docker用户组sudo groupadd docker，将当前用户加入docker组sodu usermod -aG docker $USER。默认情况下，Docker命令会使用Unix socket与Docker引擎通讯。而只有Root用户和docker组的用户才可以访问docker引擎的Unix socket。一般Linux系统上不会直接使用root用户进行操作，因此需要将使用docker的用户加入docker用户组。

  2、指令systemctl disabled firewalld禁用防火墙，通过systemctl stop firewalld关闭防火墙。

  3、指令yum remove docker docker-common docker-selinux docker-image卸载旧版本。

  4、指令yum install -y yum-utils device-mapper-persistent-data lvm2安装依赖。

  5、指令yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo增加docker下载源。

  6、指令yum install docker-ce安装docker。

# 卸载Docker

  1、指令yum remove docker-ce卸载docker。

  2、上面的指令仅仅卸载了Docker进程，但镜像还存在，使用指令rm -rf /var/lib/docker即可删除。

# 配置镜像加速

  1、入阿里云官网，搜索容器镜像服务，找到镜像加速器，复制属于自己的加速地址。

  2、指令cd /etc/docker切换目录，指令vi daemon.json编辑文件，没有就新建，并加入以下内容。

{
    "registry-mirrors": ["https://aqrlrx4a.mirror.aliyuncs.com"]
}

  3、重新加载配置systemctl daemon-reload。设置开机自启动systemctl enable docker。

# Docker-compose安装

  1、指令wget -O /usr/local/bin/docker-compose "https://github.com/docker/compose/releases/download/1.23.1/docker-compose-$(uname -s)-$(uname -m)"下载。如果提示没有wget，指令yum install -y wget安装即可。

  2、指令cd /usr/local/bin切换目录，指令mv docker-compose-Linux-x86_64 docker-compose修改名称。

  3、指令chmod +x docker-compose赋予可执行权限。

  4、指令docker-compose version看到版本号，即安装成功。
