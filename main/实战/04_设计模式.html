# 设计模式

简单工厂模式和工厂方法模式比较常用，抽象工厂的应用场景特殊。 当每个对象的创建逻辑比较简单时，推荐使用简单工厂模式，将多个对象的创建逻辑放入一个工厂类中。 当每个对象的创建逻辑比较复杂时，为了避免设计一个庞大的简单工厂类，推荐使用工厂方法模式。比如FruitFactory有具体的子类工厂，AppleFactory、BananaFactory。

为什么要用构建者模式，构建者模式可以进行更加严格的参数校验，而使用普通的set方法和构造方法却不到这样的效果。或者对象一旦创建，参数不可修改。 参数不可修改，也就是说不能暴露set方法。

public class ResourcePoolConfig {


    private String name;
    private int maxTotal;
    private int maxIdle;
    private int minIdle;

    private ResourcePoolConfig(Builder builder) {
        this.name = builder.name;
        this.maxTotal = builder.maxTotal;
        this.maxIdle = builder.maxIdle;
        this.minIdle = builder.minIdle;
    }

    private static class Builder {

        private static final int DEFAULT_MAX_TOTAL = 8;
        private static final int DEFAULT_MAX_IDLE = 8;
        private static final int DEFAULT_MIN_IDLE = 0;

        private String name;
        private int maxTotal = DEFAULT_MAX_TOTAL;
        private int maxIdle = DEFAULT_MAX_IDLE;
        private int minIdle = DEFAULT_MIN_IDLE;

        public ResourcePoolConfig build() {
            // 放到这里来做其中校验逻辑，包括必填项校验、依赖关系校验、约束条件校验等
            if (StringUtils.isEmpty(name)) {
                throw new IllegalArgumentException("name must not be empty.");
            }
            if (maxIdle > maxTotal) {
                throw new IllegalArgumentException("maxIdle must < maxTotal.");
            }
            if (minIdle > maxTotal || minIdle > maxIdle) {
                throw new IllegalArgumentException("minIdle must < maxTotal.");
            }
            return new ResourcePoolConfig(this);
        }


        public Builder setName(String name) {
            if (StringUtils.isEmpty(name)) {
                throw new IllegalArgumentException("name must not be empty.");
            }
            this.name = name;
            return this;
        }

        public Builder setMaxTotal(int maxTotal) {
            if (maxTotal <= 0) {
                throw new IllegalArgumentException("maxTotal must not < 0 .");
            }
            this.maxTotal = maxTotal;
            return this;
        }

        public Builder setMaxIdle(int maxIdle) {
            if (maxIdle <= 0) {
                throw new IllegalArgumentException("maxIdle must not < 0 .");
            }
            this.maxIdle = maxIdle;
            return this;
        }

        public Builder setMinIdle(int minIdle) {
            if (minIdle <= 0) {
                throw new IllegalArgumentException("minIdle must not < 0 .");
            }
            this.minIdle = minIdle;
            return this;
        }

        public String getName() {
            return name;
        }

        public int getMaxTotal() {
            return maxTotal;
        }

        public int getMaxIdle() {
            return maxIdle;
        }

        public int getMinIdle() {
            return minIdle;
        }
    }

    public static void main(String[] args) {

        ResourcePoolConfig resourcePoolConfig = new ResourcePoolConfig.Builder()
                .setName("connector-pool")
                .setMaxTotal(16)
                .setMaxIdle(16)
                .setMinIdle(20)
                .build();

    }

}

工厂模式来用来创建不同但是相关类型的对象，由给定的参数决定创建哪种类型的对象。 而建造者模式是用来创建一种类型的复杂对象，通过设置不同的可选参数，定制化地创建不同的对象。

public class StrategyFactory {

    private static final Map<String, Strategy> strategyMap = new HashMap<>();

    static {
        strategyMap.put("A", new StrategyA());
        strategyMap.put("B", new StrategyB());
    }

    public static Strategy createStrategy(String type) {
        if (type == null || type.length() == 0) {
            throw new IllegalArgumentException("违规的参数");
        }
        return strategyMap.get(type);
    }

    public static void main(String[] args) {

        Strategy strategy = StrategyFactory.createStrategy("");
        Assert.notNull(strategy, "未找到相关策略");
        strategy.handler();
    }

}

如果策略类是无状态的，就可以被共享使用。

回调，A调用B，B反过来调用A，这种调用机制叫做回调。

回调不仅可以应用在代码设计上，在更高层次的架构设计上也比较常见。比如通过第三方支付系统来实现支付功能，用户在发起支付请求之后，一般不会一直阻塞到支付结果返回，而是注册回调接口（类似回调函数，一般是一个回调用的URL）给第三方支付系统，等第三方支付系统执行完后之后，将结果通过回调接口返回给用户。

回调又分同步回调和异步回调，同步回调看起来像模板模式，而异步回调看起来像观察者模式。

责任链模式：让多个接收对象都有机会处理这个请求，将这些接收对象串成一条链，并沿着这条链传递这个请求，直到链上的某个接收对象能够处理它为止。

责任链有两种执行逻辑，一种是其他一个处理器能够处理那么就结束，另一种是每个处理器都要处理一遍，不存在中断的情况。

比如论坛的一些应用，用户生成的内容可能包含一些敏感词（涉黄、广告、反动等词汇）。针对这个场景，就可以利用职责链模式来过滤这些敏感词。

对于包含敏感词的内容，一种是直接禁止发布（符合GoF的定义，一旦某个处理器能够处理器这个请求，就不会继续请请求传递给后续的处理器了），另一种是给敏感词打马赛克（变种的一种方式，所有的处理器都要处理一遍）。

对于处理器有两种存储方式，一种是链表，一种是数组。
