# 优雅的读取配置文件

  通常我们会将一些配置灵活的写在配置文件中，方便于数据的更新维护。

# 读取yml配置文件

user:
  username: zhangsan
  age: 15
  address:
    country: 中国
    province: 河北省
    city: 沧州市

  通过 @ConfigurationProperties注解匹配yml配置前缀，或者直接在属性上使用 @Value("${user.username}")注解。

  通过@Configuration来让该配置类生效。而在自定义SpringBoot组件时，通常会使用@EnableConfigurationProperties(value = UserProperties.class)来让该配置类生效。

@Configuration
@ConfigurationProperties(prefix = "user")
public class UserProperties {

    private String username;

    private Integer age;

    private Address address = new Address();

    public String getUsername() {
        return this.username;
    }

    public Integer getAge() {
        return this.age;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public void setAge(Ineger age) {
        this.age = age;
    }

    public static class Address {
        private String country;
        private String province;
        private String city;

        public String getCountry() {
            return country;
        }

        public void setCountry(String country) {
            this.country = country;
        }

        public String getProvince() {
            return province;
        }

        public void setProvince(String province) {
            this.province = province;
        }

        public String getCity() {
            return city;
        }

        public void setCity(String city) {
            this.city = city;
        }
    }

}

# 读取自定义Properties文件

  找不到文件问题，看看是不是冒号与文件名之间有空格（classpath: user.properties），有空格就会报错。同时该配置类支持多个配置文件。

  如果yml中也有相同的属性配置，以yml为准。所以要避免相同的属性定义。

  如果读取中文出现乱码文件，看看properties文件的编码格式。最好在IDEA全局下定义文件为UTF-8格式，避免编码问题。

@Configuration
@PropertySource(value = {"classpath:config/user.properties"}, ignoreResourceNotFound = true)
public class UserProperties {

    @Value("${name}")
    private String name;

    @Value("${email}")
    private String email;

}
