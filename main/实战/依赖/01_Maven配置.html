    # Maven配置

    # mvn package、mvn install、mvn deploy

    这三个指令虽然都可以进行打包操作，但是是有区别的。

    • mvn package 能将项目打包，生成到target目录下。
    • mvn install 能将项目打包，生成到target目录下，并且能将jar包安装到本地仓库。
    • mvn deploy 能将项目打包，生成到target目录下，并且能将jar包安装到本地仓库，还能上传到远程仓库Nexus。

    # 如何将一个sdk安装到本地仓库

    mvn install:install-file -Dfile=E:\is-security-cas-1.0.0.jar -DgroupId=kuaishou -DartifactId=is-security-cas -Dversion=1.0.0 -Dpackaging=jar

      IDEA版本与Maven存在版本冲突问题，本文基于IDEA2019.2版本，要求安装Maven3.6.1以下版本。Maven下载地址，下载合适版本的binaries即可。

      修改conf/Settings.xml文件配置。配置如下：

    <?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0 http://maven.apache.org/xsd/settings-1.0.0.xsd">
  
	<!-- 设置 Maven 仓库存放地址 -->
	<localRepository>D:\DevelopSoftware\apache-maven-3.6.1\repository</localRepository>

	<mirrors>
		<!-- 配置阿里云镜像仓库地址 -->
		<mirror>
			<id>alimaven</id>
			<name>aliyun maven</name>
			<url>http://maven.aliyun.com/nexus/content/groups/public/</url>
			<mirrorOf>central</mirrorOf>
		</mirror>
	</mirrors>
	<profiles>
	</profiles>
	<pluginGroups>
	</pluginGroups>
	<proxies>
	</proxies>
	<servers>
	</servers>
</settings>

      配置环境变量。在系统变量中，声明一个变量MAVEN_HOME，值为D:\DevelopSoftware\apache-maven-3.6.1。在Path变量中，增加一段值%MAVEN_HOME%\bin;即可。

    # 通过POM文件修改仓库地址

      上面是对仓库的全局配置，也可以通过pom的方式来配置阿里云镜像仓库。

    <repositories>
	<!-- 阿里云镜像仓库 -->
    <repository>
        <id>central</id>
        <name>aliyun maven</name>
        <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
        <releases>
            <enabled>true</enabled>
        </releases>
        <snapshots>
            <enabled>false</enabled>
        </snapshots>
    </repository>
	<!-- 私服镜像仓库 -->
	<repository>
        <id>nexus</id>
        <name>Nexus Repository</name>
        <url>http://192.168.29.202:8080/repository/maven-public/</url>
        <releases>
            <enabled>true</enabled>
        </releases>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>

    # 打包方式

    1. mvn package 将项目打包。

    2. mvn install 将项目打包，并安装到本地仓库。

    3. mvn deploy 将项目打包，并安装到本地仓库以及上传到Nexus私服。

    # 项目瘦身

      项目中的依赖一般是不会变的，改变的都是代码而已，每次重新编译打包长传服务器，由于存在很多依赖，所以上传时间很久。

      使用如下配置进行打包，将第一次打包后的jar包解压出来，将其中lib文件夹复制出来。

    <build>
	<plugins>
		<plugin>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-maven-plugin</artifactId>
			<configuration>
				<mainClass>com.tellyes.stormx.validator.ValidatorApplication</mainClass>
				<layout>ZIP</layout>
			</configuration>
			<executions>
				<execution>
					<goals>
						<goal>repackage</goal>
					</goals>
					<configuration>
						<classifier>exec</classifier>
					</configuration>
				</execution>
			</executions>
		</plugin>
		<plugin>
			<groupId>org.apache.maven.plugins</groupId>
			<artifactId>maven-compiler-plugin</artifactId>
			<configuration>
				<source>8</source>
				<target>8</target>
				<encoding>UTF-8</encoding>
			</configuration>
		</plugin>
		<plugin>
			<groupId>org.apache.maven.plugins</groupId>
			<artifactId>maven-resources-plugin</artifactId>
			<configuration>
				<encoding>UTF-8</encoding>
			</configuration>
		</plugin>
	</plugins>
</build>

      当修改了代码，再次编译打包时使用以下配置。

    <build>
	<plugins>
		<plugin>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-maven-plugin</artifactId>
			<configuration>
				<mainClass>com.tellyes.stormx.validator.ValidatorApplication</mainClass>
				<layout>ZIP</layout>
				<!-- 第二次打包时加上这段配置 Begin -->
				<includes>
					<include>
						<groupId>nothing</groupId>
						<artifactId>nothing</artifactId>
					</include>
				</includes>
				<!-- 第二次打包时加上这段配置 End -->
			</configuration>
			<executions>
				<execution>
					<goals>
						<goal>repackage</goal>
					</goals>
					<configuration>
						<classifier>exec</classifier>
					</configuration>
				</execution>
			</executions>
		</plugin>
		<plugin>
			<groupId>org.apache.maven.plugins</groupId>
			<artifactId>maven-compiler-plugin</artifactId>
			<configuration>
				<source>8</source>
				<target>8</target>
				<encoding>UTF-8</encoding>
			</configuration>
		</plugin>
		<plugin>
			<groupId>org.apache.maven.plugins</groupId>
			<artifactId>maven-resources-plugin</artifactId>
			<configuration>
				<encoding>UTF-8</encoding>
			</configuration>
		</plugin>
	</plugins>
</build>

      使用命令java -Dloader.path=D:\project\StormX\storm-validator\target\lib -jar storm-validator-1.0-SNAPSHOT-exec.jar即可通过外置依赖启动项目。
